IUB Digital Campus - IT Admin (Main Controller) Dashboard Specification
======================================================================

1. GOAL
=======
The IT Admin (Main Controller) Dashboard is the central hub for IT personnel to manage user provisioning, security, backups, logs, and system integrations. It provides a comprehensive, role-based interface that allows IT admins to:
- Provision and manage user accounts across the entire system
- Monitor and maintain system security
- Manage data backups and recovery procedures
- Analyze system logs and performance metrics
- Handle system integrations with external services
- Select advisors for students
- Monitor overall system health and performance
- Respond to technical incidents and issues

The dashboard serves as the "Main Controller" for the entire digital campus platform, ensuring the stability, security, and performance of all system components. It emphasizes the critical role of IT admins in user provisioning and advisor selection.

5. FUNCTIONS TO AVOID ON MOBILE (FOR OPTIMIZED PERFORMANCE)
===========================================================
To ensure optimal performance on mobile devices and maintain security for critical administrative functions, the following functions should be limited or avoided on mobile:

5.1. Resource-Intensive Features
--------------------------------
- Real-time system health analytics with complex visualizations
- Large log file analysis and rendering
- Complex report generation (should be deferred to desktop)
- Bulk user provisioning operations
- Simultaneous multiple system transactions
- Infrastructure monitoring dashboards

5.2. Bandwidth-Heavy Features
-----------------------------
- High-frequency polling for system notifications (should use push notifications)
- Automatic loading of large system datasets
- Background synchronization of extensive log data
- Real-time collaborative editing of system configurations

5.3. Processing-Intensive Features
----------------------------------
- Advanced data visualization rendering
- Machine learning-based threat detection
- Real-time chat translation services
- Complex system optimization algorithms

5.4. Security-Sensitive Features (Must be done on PC)
----------------------------------------------------
- User provisioning and account management (system-wide critical function)
- Advisor selection for students (critical academic function)
- Security settings modification
- System backup and recovery operations
- Access control configuration
- Authentication system management
- Encryption key management
- System integration modifications
- Disaster recovery procedures
- Audit trail management
